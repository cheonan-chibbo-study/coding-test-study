# 문제 링크
https://www.acmicpc.net/problem/19238

# 접근 방법
문제에서 요구하는 바는, 택시가 현재 위치에서 가장 가까운 승객을 태워서 해당 승객의 목적지까지 이동하는 과정을 반복해서 모든 승객을 태우거나, 중간에 연료가 다 떨어지거나, 더이상 태울 승객이 없을 때까지 반복했을 때 마지막에 남은 기름의 양입니다.

2차원 격자 상에서 한 좌표에서 다른 좌표까지의 최단 거리는 BFS를 통해 쉽게 구할 수 있습니다.  
그리고 사실상 문제에서 요구하는 바를 잘 **구현만** 하면 되기에 특별한 접근 방법은 없었습니다.  

그래도 문제의 조건을 꼼꼼히 살펴보려고 했습니다. 코드 작성 이전에는 주석으로 해결 단계를 잘게 나누어 작성했고 이를 기반으로 메서드를 분리하여 구현했습니다.  
하지만 구현을 하다보니 메서드 간 값 전달이 필요했고 단순한 구현을 위해 `drive()`와 `move_to_goal()` 이라는 두 개의 메서드만이 남게 되었습니다.

1시간 이내에 모든 구현을 완료했지만, 약 84%에서 실패했습니다.  
이미 시간이 많이 지체되기도 했고 디버깅 할 자신이 없어 AI의 도움을 받았습니다.

실수한 부분은 다음과 같습니다.
- 다음 승객까지 이동할 수 없는 상황임에도 이동이 가능하게 구현되었다.
- 목적지까지 이동할 수 없는 상황임에도 이동이 가능하게 구현되었다.

복잡한 시뮬레이션 문제의 경우 주어진 예시에만 의존하여 구현하는 경향이 있는 것 같습니다.  
그래서 이런 실수를 하지 않았나 싶습니다.
