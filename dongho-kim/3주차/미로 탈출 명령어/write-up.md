# 접근 방법
시작 위치부터 가능한 모든 경로를 BFS 방식으로 탐색합니다.

하지만 이렇게 되면 정답을 찾더라도 불필요한 탐색과 중복 탐색이 지나치게 많아집니다.

따라서 특정 (x, y) 위치에 방문을 할 때, 불필요한 연산을 제거하는 가지치기가 중요합니다.

불필요한 연산은 다음과 같습니다.

1. 격자를 벗어난 경우
2. 탐색하려는 위치로 탐색했을 때, (r, c)까지 남은 거리가 이동 가능한 거리보다 더 큰 경우
3. 탐색하려는 위치에 이미 같은 길이로 탐색을 이미 한 경우 -> 애초에 문제에서 요구하는 대로 알파벳 순서대로 탐색을 하면, 똑같은 횟수로 똑같은 위치를 더이상 탐색할 필요가 없습니다.

# 배운 점
1. BFS 유형도 경우에 따라 가지치기가 정말 중요하다.
2. BFS 유형은 결국 `while`문 안에서 계속해서 탐색하는 곳을 찾는 로직이 전부라고 느껴진다.
