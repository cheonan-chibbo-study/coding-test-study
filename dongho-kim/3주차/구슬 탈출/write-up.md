# 문제 링크
https://www.acmicpc.net/problem/13459

# 접근 방법
처음에는 옛날에 시뮬레이션 유형 연습할 때 좌표 땡기기 기술을 배운 적이 있고, 이 문제에서 쓰면 적합하겠다 싶어서 BFS 말고 시뮬레이션으로만 구현하려고 했습니다.

그러다보니 시뮬레이션으로 구현을 어떻게 해야 할 지 감이 잘 오지 않았습니다.

그래서 BFS를 이용해서 풀 생각을 했습니다.

구슬의 위치를 큐에 넣고, 각 구슬의 위치를 기준으로 네 방향을 모두 이동해보는 경우를 따져보았습니다.  
이때 구슬이 겹치게 되는 경우가 있는데, 이는 이동 거리가 더 긴 구슬을 한 칸 뒤로 보정하는 로직을 추가했습니다.

BFS로 접근하니 아이디어는 쉽게 정리가 됐지만, 구현하는 부분에서 쉽지 않은 문제였습니다.

# 배운 점
1. 아직 DFS/BFS를 이용해서 푸는 게 익숙하진 않다. 특히 BFS의 경우 조금만 복잡해져도 머리 속에서 흐름이 잘 그려지지가 않는 것 같다.
