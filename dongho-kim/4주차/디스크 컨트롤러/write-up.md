# 접근 방법
문제에서 요구하는 바는 모든 요청 작업의 반환 시간의 평균입니다.

이를 구하기 위해서 일단 요청이 들어온 작업 순서대로 정렬한 다음, 현재 시각(`time`)에 따라 우선순위 큐에 넣습니다.

그 다음, 우선순위 큐에서 작업을 추출해서 해당 작업의 반환 시간을 구합니다.

---

맨 처음 구현할 때는 모든 작업을 (작업의 소요시간이 짧은 것, 작업의 요청 시각이 빠른 것, 작업의 번호가 작은 것) 순으로 우선순위 큐를 만든 다음, 각 작업을 하나씩 빼면서 시간을 측정했었는데요.

이 방법은 예를 들어 현재 시각(`time`)이 0임에도 불구하고 나중에 들어올(예: `time` 10) 짧은 작업을 먼저 수행하려 합니다. 따라서 대기 큐에는 현재 시간까지 이미 요청이 들어온 작업들만 들어가야 합니다.

# 배운 점
1. 우선순위 큐를 사용하는 것과 작업 반환 시간을 구하는 것은 어렵지 않았지만, 문제의 디테일한 요소를 놓쳤습니다. 문제를 조금 빠르고 대충 읽은 탓인 것 같습니다.
