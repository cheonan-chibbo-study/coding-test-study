# 문제 링크

https://leetcode.com/problems/subsets/

# 접근 방법

재귀함수로 `nums`를 순회하면서 숫자를 하나씩 뽑아야겠다고 생각했습니다.  
그런데 `nums`의 길이는 고정되어 있지 않아서 재귀함수를 작성해야겠다고 생각했습니다.

재귀 함수를 작성하기 전, 문제 요구사항을 자세히 살펴보니 숫자를 뽑을 때 0개부터 `nums` 사이즈만큼 뽑아야 하며, 동시에 인덱스를 하나씩 올려가면서 뽑는 규칙임을 발견했습니다.  
이를 기반으로 재귀함수를 작성했습니다.

각 재귀함수에서는 `start_index`부터 `nums` 마지막 인덱스까지 순회하면서 숫자를 하나씩 뽑는데, 이때 현재까지 뽑은 `result`에 숫자를 추가로 뽑을 경우 `target_size`보다 커질 경우
뽑는 연산은 불필요하므로 제외했습니다.

종료 조건은 뽑은 숫자를 저장하는 `result` 리스트의 크기가 주어진 `target_size`에 도달할 경우 `answer`에 뽑은 숫자 조합을 기록하고 종료하는 식으로 구현했습니다.

재귀 함수는 주어진 `target_size`만의 수를 뽑는 데 집중했고, `target_size`는 메인 메서드에서 정해지도록 구현했습니다.
