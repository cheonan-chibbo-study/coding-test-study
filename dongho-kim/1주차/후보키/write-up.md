# 접근 방법
일단 모든 속성 조합을 구하고 그 조합이 유일성과 최소성을 만족하는 지 여부를 판단하는 완전탐색 방식으로 접근했습니다.

이를 재귀함수로 구현했습니다.  
모든 i번 인덱스(속성)에 대해 선택하거나 선택하지 않는 모든 경우를 따지고, 모두 선택했을 때 종료부에서 조건에 따라 유일성과 최소성을 만족하는 지 체크했습니다.

# 배운 점
1. 문제 덕분에 후보키에 대해 좀 더 확실히 이해했습니다.
2. `solution()` 안에서 `answer` 변수를 초기화 후, `recurse()`로 전달해서 해당 메서드 안에서 갱신해줬는데, Python Scope 개념 때문에 `solution`에 선언된 `answer`이 아닌, `recurse()`에서 `answer`이 새롭게 선언되고 있었습니다. 메서드 파라미터로 전달하면 참조값이 전달되니까 반영이 될 줄 알았는데 안됩니다. 좀 더 생각해보니 `answer += 1` 은 `answer = answer + 1`이고, 이때 우측 항의 `answer`은 메서드 인자로 전달되는 `answer`이 맞습니다. 그런데 `=`에 의해 새롭게 `answer` 이라는 변수에 값을 할당하는 순간, 새로운 지역 변수 `answer`를 생성하는 방식으로 동작합니다. 따라서 `answer`를 전역 변수로 승급시켜서 처리해줬습니다. Python의 LEGB가 아직은 익숙치 않네요.
