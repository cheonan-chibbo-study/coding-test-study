# 문제 링크

https://leetcode.com/problems/binary-search/

# 접근 방법

1. 완전탐색
   for문으로 nums 순회하면서 `target`을 찾는 방법을 적용해봤다.  
   `nums`의 최대 길이가 10^4라서 시간이 오래 걸리진 않지만, 문제에서 요구하는 시간 복잡도는 `O(logN)` 였다.

2. 이진탐색
   문제 이름에서 대놓고 힌트를 줘서 이진탐색을 구현했다.  
   while 조건문이 기억나지 않아 처음엔 무한 루프를 돌렸는데, 시간 초과가 발생해서 로그를 찍어보니 l이 r보다 커진 경우가 발생했다.  
   이걸 보고 기억나서 `l <= r`로 설정하니 통과했다.

# 배운 점

1. 이진탐색에서 l이 r보다 커지면 탐색하지 않는다.
