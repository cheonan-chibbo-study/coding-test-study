# 문제 링크

https://leetcode.com/problems/word-search/description/

# 접근 방법

1. `board`를 모두 순회하면서 `word`의 첫 글자에 해당하는 요소를 가진 위치라면, 해당 위치에서부터 DFS 탐색을 시작한다.
2. 상하좌우에 있는 값들을 하나씩 보면서, 격자 내부에 포함되면서 다음 index에 해당하는 값이 있고 아직 탐색하지 않은 칸인 경우에만 다음 칸으로 이동한다.
3. `word`의 길이만큼 탐색을 수행했다면, `word`가 `board` 내에 존재한다는 의미로 성공 처리를 하고 탐색을 종료한다. 만약 탐색에 실패하는 경우에는 별다른 처리없이 DFS 탐색을 바로 종료한다.
4. 마지막에 성공 여부 처리를 반환한다.

# 배운점

1. 문제를 대충 파악하고 바로 구현을 시작했다. 그래서 처음에 설계를 잘못해서 시간을 허비했다. 문제를 좀 더 꼼꼼히 파악하고, 이를 기반으로 설계를 좀 더 했더라면 어땠을까 싶다.
2. 중첩 함수(`dfs()`)에서 부모 함수(`exist()`) 내 지역 변수에 접근하는 과정에 의도한대로 동작하지 않았다. 찾아보니 중첩 함수에서 부모 함수의 지역
   변수에 접근하려면 `nonlocal` 키워드로 먼저 선언해줘야 된다는 것을 알게 됐다.
