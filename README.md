# 천안 오프라인 코딩 테스트 스터디

## 스터디 목표
1. 코테 전형 뚫기
2. 매일 꾸준하게 두 문제 풀기

## 스터디 방식
### 오프라인 스터디 하기 전
- 매일 정해진 두 문제를 풀어서 풀이 코드와 write-up을 스터디 레포지토리에 푸시
  - 해당 문제를 어떤식으로 접근하고 풀었는지 기술하는 `write-up.md` 필수
  - 풀이 코드를 따로 작성해서 푸시 (ex. `solution.py`)
- 하루라도 두 문제 풀이 & 저장소 푸시를 실패하면 그 주차 오프라인 스터디에 드는 비용을 부담
    - 실패자가 1명이면 그 1명이 전부 부담
    - 실패자가 n명이면 그 n명이 나눠서 부담

### 문제 풀이 인증
문제 풀이 인증 저장소는 아래 폴더 구조에 맞춰 `write-up`과 풀이 코드 작성
```
kelly (본인 이름)
└── 1주차 (현재 주)
    └── 피로도 (노션에 적힌 문제명)
        ├── solution.py
        └── write-up.md
```

인증 커밋은 아래 양식을 맞춰서 Push
```
(현재 주) {문제명} 풀이 인증

ex. (1주차) 피로도 풀이 인증
```

저장소 Push는 본인 폴더명으로 브랜치 생성 후 PR 작성해서 `main`에 merge
- PR 제목 양식은 `(주차) {오늘 날짜} {본인 본명} 문제 풀이 인증`
  - ex. `(1주차) 2026.01.28 양성욱 문제 풀이 인증`
- PR 본문은 자유롭게 작성

### 오프라인 스터디
- 오프라인 스터디는 매주 화요일 2시간 진행
    - 매주 화요일 16:00
    - 장소는 신부동 신세계 백화점 근처 스터디룸 대여
- 1시간은 면접 스타일로 진행
  - 1명이 면접자, 나머지는 면접관 역할을 맡으며 돌아가면서 진행 (한 사이클 당 20분)
  - 면접관은 문제 하나를 선정해서 면접자에게 질문하고 면접자는 답변
    - `Hard`, `Lv.03`, `Gold` 난이도의 문제로 선정
    - 문제 접근 방식, 풀이를 선택한 이유 등의 질문을 나눔
- 나머지 1시간은 각자 문제 풀이 방식으로 진행
  - 스터디원 전원 풀어보지 못한 문제를 매주차 선정 후 스터디 당일 하나를 선택해 당일 풀이
  - 실제 코테 응시 환경과 비슷하게 설정해서 문제 풀이 진행 (제힌시간, 에디터 사용 금지 등등)
  - 문제 풀이 후 서로 피드백 공유
    - 문제를 시간안에 풀었다면 어떻게 풀었는지
    - 문제를 시간안에 풀지 못했다면 왜 못풀었는지

## 스터디 규칙
- 학습 목적의 스터디인 만큼 가끔 스몰톡은 진행해도 스터디 본질에 집중합니다.
- 경쟁이 목적이 아니므로 서로 격려하고 도와주며 나아갑시다.
- 패널티를 받지 않기 위해 꼼수를 쓴다면 얼마든지 쓸 수 있겠지만 그건 결국 본인 손해입니다. 다들 2026년은 꼭 취업하고 싶은 간절함에 모인 만큼 취뽀를 위해 열심히 해봅시다.
