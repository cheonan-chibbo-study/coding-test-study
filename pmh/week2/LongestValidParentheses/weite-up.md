# 접근방법
1. 괄호에 관한 문제라서 스택을 쓰면될것같다고 생각해서 deque를 사용
2. 괄호가 () 인 갯수를 모두 구하는거는 성공했지만 가장긴 () 찾는게 어려워서 힌트를좀 찾아봄
3. 문자열 값대신 인덱스를 스택에 저장한다
   - ( 를 만나면 스택에 현재 인덱스 푸쉬
   - ) 를 만나면 현재 인덱스 - 스택에 들어있던 인덱스
4. 기준점(BaseIndex) 설정
   - 유호한 괄호 길이를 잴 때 어디서 부터 시작 되었는지 알려줄 기준이 필요함
   - 처음 스택에 -1 넣고 시작해보기 -1 아직 아무것도 시작되 않은 지점 혹은 마지막으로 유효하지 않았던 괄호의 위치를 의미하게된다
5. 길이 업데이트 시점
   - )를 만나서 스택에서 pop 을 한직후가 길이를 잴 타이밍
   - pop 를 한후 스택이 비어있지않다면 현재 인덱스 - 현재 스택의 top 에 있는 값이 현재까지 유효한 길이 이다
   - 만약 스택이 비어있다면 현재 ) 의 인덱스가 새로운 기준점이 되므로 ) 의 인덱스 를 push 한다
