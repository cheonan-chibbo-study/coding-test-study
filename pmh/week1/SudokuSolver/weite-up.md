# 접근방법
문제 접근 방법이 떠오르지 않아 해설을 참고해서 풀었다
1. 어떤 알고리즘을 사용할까? : 백트래킹
   - 수도쿠 의 모든 칸을 탐색해야하니 완전 탐색을 처음에 생각했다. 그러나 완전 탐색 사용시 81개의 모든 빈칸에 1~9 까지 숫자를 하나 씩 넣어보게 되므로 
   경우의수가 9^81 가 되어 컴퓨터가 1초에 1억번(10^8) 계산한다고해도 시간 초과가 난다
   - 그래서 모든경우의 수가 아닌 가능서 있는 것만 확인하는 백트래킹을 사용한다. 백트래킹의 핵심은 가다가 아니면 돌아온다 이다 즉 숫자를 넣어보다가 스도쿠 규칙에 어긋나면 그 뒤는 더이상
   탐색하지않고 포기(가치지기) 한다
   - 이로 인해 수도규 가로 규칙,세로 규칙, 3x3 규칙 확인을 총 81 번 한다고하면 9^3 * 9^2 = 9^5 가 되므로 시간초과가 나지않을것이라고 생각했다.

2. 구현방식
   - row = s/9 col = s % 9 를 통해 현재 탐색하는 칸의 행과 열을 구한다.
   - board[row][col] 에 '.' 아니라 숫자가있다면 s+1 하고 다음 백트래킹 함수를 호출한다
   - 만약 '.' 라면 '1'<=c<='9' 까지 넣어가면서 해당칸을 탐색한다
   - 탐색할때마다 isValid 를 통해 해당칸에 극 숫자가 들어가되는지 체크한다
     - isValid 에 칸의 행,열,숫자 를 넘겨줘 전체 행 과 열 에 그 숫자기 있는지 확인하다 있으면 false 반환
     - board[3 * (row /3) +i/3][3 * (col /3) +i%3] 를 통해서 해당칸에 3x3 격자 의 시작점을 찾아 탐색한다 그 숫작 있다면 false 반환
     - 다 통과시 ture 반환
   - 들어갈수있으면 해당칸에 해당 숫자를 넣고 다시 s+1 하고 백트래킹 호출 만약 탐색하다 중복이 나오면 해당칸을 지우도로 '.' 을추가한다
   - 만약 모든 포문 통과시 탐색완료로 true 반환한다

# 배운점
1. 해당 칸을 보드 크기로 /, % 할시 해당 칸의 행과 열을 알수 있다.
```java
int row = s/9;
int col = s%9;

[예]
s:17 -> (1,8)
        9*9 격자내의 17번째 셀의 위치는 (1,8)이다.
```
 9 x 9 보드 안에 각 3 x 3 보드의 시작점 찾는법
   - 3 * (row/3) ,3 * (col /3)
     - row/3 을 하면 0, 1, 2 는 0 이되고 3, 4, 5 이는 1 되고 6, 7, 8 은 2가된다(몇번째 큰 덩어리 행인지 구할수있음) 열도 똑같음
     - 여기에 3 * 을 하면 0, 3, 6 이된다 해당 구영의 시작인덱스 가 된다
   - i / 3 과 i % 3
     - 이 부분의 3 x 3 의 시작 부분터 오른쪽, 아래로 조금씩 이동해 9칸을 탐색하는 방식이다 

   |  i 값  | i / 3 (행 이동) | i % 3 (열 이동) | 위치 설명        |
   |:-----:|:------------:|:------------:|:-------------|
   | **0** |      0       |      0       | 시작점 (좌상단)    |
   | **1** |      0       |      1       | 오른쪽 1칸       |
   | **2** |      0       |      2       | 오른쪽 2칸       |
   | **3** |      1       |      0       | **다음 줄** 첫 칸 |
   | **4** |      1       |      1       | 가운데          |
   | **5** |      1       |      2       | 오른줄 2칸       |
   | **6** |      2       |      0       | 다음줄 첫칸       |
   | **7** |      2       |      1       | 오른쪽 1칸       |
   | **8** |      2       |      2       | 끝점(우하단)      |
