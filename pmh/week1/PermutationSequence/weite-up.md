# 접근방법

1. n 이 주어지면 1~n 까지의 순열의 수열 만들고 k 번째의 순열을 반환하는 문제였음
2. n 이 최대 9여서 수가 별로 크지않아 dfs 로 가능한 모든 순열 을 만든후 그중 k번째를 반환해주는 방식으로 짜면 될것같다고 생각함
3. 또한 dfs 방식으로 하면 탐색하면서 점점 커지는 수열이되므로 더욱 적합하다고 판단했음

# 접근방법2
1. 위의 방법은 n 이 커지면 시간을 초과하는 경우가 생길수있음 그래서 검색후 또다른 방법을 찾음
2. 팩토리얼 진법을 사용한다
   1. 핵심 아이디어: "내 앞에 몇 덩어리가 있는가? "$n=3$ 일 때 순열을 사전순으로 나열해본다.
      - 1로 시작하는 순열: [1, 2, 3], [1, 3, 2] (총 $(3-1)! = 2$개)
      - 2로 시작하는 순열: [2, 1, 3], [2, 3, 1] (총 $(3-1)! = 2$개)
      - 3로 시작하는 순열: [3, 1, 2], [3, 2, 1] (총 $(3-1)! = 2$개)
      - 만약 $k=3$ (3번째 순열)을 찾고 싶다면:
      - 한 숫자로 시작할 때마다 뒤에 2개씩($2!$) 묶음이 생긴다.
      - $k=3$이라면, 앞의 '1로 시작하는 묶음(2개)'은 이미 지나갔다는 뜻이다.
      - 따라서 첫 번째 숫자는 2가 된다.
   2. 계산 단계 (n=4, k=9 예시)사용 가능한 숫자: [1, 2, 3, 4], $k=9$ (인덱스로 치면 $k-1 = 8$)
      - [Step 1] 첫 번째 자리 결정
        - 첫 자리를 고정하면 뒤에 $3! = 6$개의 경우의 수가 생긴다.
        - $8 \div 6 = \mathbf{1}$ (몫), 나머지 2
        - 숫자 리스트 [1, 2, 3, 4]에서 인덱스 1인 숫자 2를 선택한다.
        - 남은 숫자: [1, 3, 4], 새로운 $k = 2$
      - [Step 2] 두 번째 자리 결정
        - 남은 자리는 2자리이므로 뒤에 $2! = 2$개의 경우의 수가 생긴다.
        - $2 \div 2 = \mathbf{1}$ (몫), 나머지 0
        - 숫자 리스트 [1, 3, 4]에서 인덱스 1인 숫자 3을 선택한다.
        - 남은 숫자: [1, 4], 새로운 $k = 0$
      - [Step 3] 세 번째 자리 결정
        - 뒤에 $1! = 1$개의 경우의 수가 생긴다.
        - $0 \div 1 = \mathbf{0}$ (몫), 나머지 0
        - 숫자 리스트 [1, 4]에서 인덱스 0인 숫자 1을 선택한다.
        - 남은 숫자: [4], 새로운 $k = 0$
      - [Step 4] 네 번째 자리 결정
        - 마지막 남은 숫자 4를 선택한다.결과: "2314"
