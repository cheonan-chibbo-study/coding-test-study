# 문제링크

https://leetcode.com/problems/word-search/description/

# 접근방법
1. 문제 분석 및 탐색 전략
   - 완전 탐색 필요성: 보드 내에서 단어의 모든 글자가 연속해서 존재하는지 확인해야 하므로 보드 전체를 탐색하는 완전 탐색이 필요함.
   - 탐색 방식: 보드의 각 칸을 시작점으로 삼아 단어를 완성할 수 있는지 확인해야 하므로 DFS(깊이 우선 탐색) 방식을 채택함.
2. 시작점 설정 및 재귀 구현
   - 시작점 찾기: 이중 for문을 사용하여 보드의 각 좌표(r, c)를 순회하며, 단어의 첫 글자(word.charAt(0))와 일치하는 지점을 찾음.
   - 재귀적 탐색: 시작점을 찾으면 해당 위치에서 상하좌우 네 방향으로 재귀 함수를 호출함. 이때 탐색의 단계를 나타내는 index를 1씩 증가시켜 단어의 다음 글자와 보드의 문자를 대조함.
3. 경계 조건 및 예외 처리
   - 성공 조건: index가 찾고자 하는 단어의 길이와 같아지면 모든 글자를 순서대로 찾은 것이므로 true를 반환함.
   - 범위 및 일치 여부 체크: 재귀 호출 시 아래 조건에 해당하면 false를 반환하여 탐색을 종료함.탐색 중인 좌표(r, c)가 보드의 범위를 벗어나는 경우 ($r < 0, r \ge n$ 또는 $c < 0, c \ge m$).현재 보드의 문자가 단어의 현재 인덱스 문자와 일치하지 않는 경우
4. 방문 처리와 백트래킹 (Backtracking)
   - 방문 처리: 한 번의 단어 완성 과정에서 같은 칸을 중복 사용하는 것을 방지하기 위해, 방문한 칸은 임시 문자(예: '#')로 변경하여 표시함.상태
   - 복구: 특정 경로의 탐색이 끝나면(성공하지 못하고 돌아올 때), 다른 경로에서 해당 칸을 다시 사용할 수 있도록 원래 문자로 복구하는 과정이 필수임.
