# 접근방법
 관계 데이터베이스에서 릴레이션(Relation)의 튜플(Tuple)을 유일하게 식별할 수 있는 속성(Attribute) 또는 속성의 집합 중, 다음 두 성질을 만족하는 것을 후보 키(Candidate Key)라고 한다. 
 - 유일성(uniqueness) : 릴레이션에 있는 모든 튜플에 대해 유일하게 식별되어야 한다. 
 - 최소성(minimality) : 유일성을 가진 키를 구성하는 속성(Attribute) 중 하나라도 제외하는 경우 유일성이 깨지는 것을 의미한다. 즉, 릴레이션의 모든 튜플을 유일하게 식별하는 데 꼭 필요한 속성들로만 구성되어야 한다.
후보키 최대 개수 구하기

제한사항 
- relation은 2차원 문자열 배열이다. 
- relation의 컬럼(column)의 길이는 1 이상 8 이하이며, 각각의 컬럼은 릴레이션의 속성을 나타낸다. 
- relation의 로우(row)의 길이는 1 이상 20 이하이며, 각각의 로우는 릴레이션의 튜플을 나타낸다. 
- relation의 모든 문자열의 길이는 1 이상 8 이하이며, 알파벳 소문자와 숫자로만 이루어져 있다. 
- relation의 모든 튜플은 유일하게 식별 가능하다.(즉, 중복되는 튜플은 없다.)

1. 가능한 모든 경우의 수를 탐색해야하니 dfs 를 사용해야한다고 생각함
2. dfs 탐색해서 가능한 모든 조합을 구한다 
3. 최소성을 쉽게  식별하기 쉽게하기위해서 조합을 오름차순(짧은 순) 으로 정렬한다
   - 만약 [0] 키 가 이미 후보키로 등록 되어있다면 [0,1] 은 이미 최소성을 만족하지못하므로 후보키가 되지못하기때문이다.
4. 또한 최소성을 만족하지 못한다면 유일성은 만족하지 못하기때문에 최소성을 먼저 검사하고 유일성을 검사한다.

```java

// candidates: 이미 합격해서 후보키가 된 녀석들 (예: [[0]])
// current: 지금 심사받으러 온 조합 (예: [0, 1])

for (List<Integer> key : candidates) {
    // containsAll: current 리스트 안에 key 리스트의 요소가 '전부' 들어있는가?
    if (current.containsAll(key)) {
        return false; // 들어있다면 너는 최소성 탈락이야!
    }
}
```
최소성의 정의 : 내가 후보키가 되려면, 이미 후보키로 선정된 녀석들을 내안에 포함하고 있으면 안된다

1. 조합을 길이가 짧은 순더래오 정렬해서 검사했음
    - 그래서 candidated 리스트에는 항상 cureent 보다 길이가 같거나 작은 키들만 들어있음
2. [0] 인 학번은 이미 찾은 후보키 임 -> 지금 검사할 조합(current) 인 [0,1]
3. containsAll 를 통해 [0,1] 안에 [0] 은 모든 내용물들이 들어있는지 검사 -> YES
4. [0] 만 있어도 식별이 가능한데 1 이 붙어 있으니 최소한의 구성아니 아니므로 최소성 탈락으로 false 반환
