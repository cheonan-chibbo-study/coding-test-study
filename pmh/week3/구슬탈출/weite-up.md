# 문제링크

https://www.acmicpc.net/problem/13459

# 접근방법
1. 큐 에 담을때 blue, red 확인하는 변수를 추가하고 bfs 를 하면 될것같다고 생각했다
2. 그러나 이렇게할시 blue red 가 동시에 움직이는게 아닌 따로 움직이는 방식으로 되서 실패했다. 또한 문제를 잘못이해해서 기울여서 벽이 만날때까지 이동인데 한번 기울이면 한번 이동하는걸로 착각해 문제푸는데 더 시간이걸렸다.
3. 그래서 레드 블루 공을 동시에 저장하도록 state 클래스 만들어서 저장하고 visited을 4차원 배열로 선언해서 레드 , 블루 공이 간경로를 동시에 알수있게했다

# 배운점 
 - 두개를 동시에 움직이거나할때 4차원 배열을 쓸수도 있다는것을 알게됬다
 - 문제를 정확히 이해해야겠다고 생각했다.
 - 클래스를 써야겠다고 생각이 드는게 쉽지않는걸 느꼈다.
